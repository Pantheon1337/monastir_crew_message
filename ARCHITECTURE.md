# Monastir Crew Message — архитектура

## Цели продукта

Социальная сеть с упором на **сырой минимализм**: чаты (личные, группы, публичные комнаты), лента постов **без видимых лайков**, сторис с круглыми аватарками и прогрессом просмотра. Реалтайм через **WebSocket**; REST — для CRUD и пагинации.

## Высокоуровневая схема

```
       └───────────────┬───────────────────┘
                       │
              ┌────────▼────────┐
              │ WebSocket server │
              │ (тот же процесс) │
              └─────────────────┘
```

## Доменные сущности

| Сущность | Назначение |
|----------|------------|
| **User** | Профиль, аватар, статус онлайн |
| **Story** | Набор слайдов (или одно фото/текст), TTL, просмотры |
| **Post** | Текст/медиа в ленте; лайки могут существовать на сервере для ранжирования, но **в UI не показываются** |
| **Chat** | Тип: `direct` \| `group` \| `room` (паблик-канал) |
| **Message** | Принадлежит чату; для комнат — видны подписчикам |
| **Room** | Публичная комната: имя с `#`, участники, last active |

## Каналы real-time (WebSocket)

События в одном JSON-формате: `{ type, payload }`.

| Тип | Назначение |
|-----|------------|
| `presence:update` | Кто онлайн (для точек у аватаров) |
| `typing` | Индикатор набора в чате |
| `message:new` | Новое сообщение в подписанных чатах |
| `story:view` | Синхронизация прогресса сторис (опционально) |
| `room:activity` | Обновление «last active» для комнат |

Клиент подписывается на комнаты/чаты после аутентификации (в базовой версии — `userId` из query или заголовка; в проде — JWT).

## REST (черновик маршрутов)

- `GET /api/health` — живость сервиса  
- `GET /api/feed` — лента постов  
- `GET /api/chats` — превью чатов  
- `GET /api/rooms` — список комнат  
- `GET /api/stories` — кольцо сторис  

Дальше: `POST` для сообщений, постов, создания комнат — по мере подключения БД.

## Фронтенд (компоненты)

- `Header` — заголовок, поиск, меню  
- `StoriesBar` — горизонтальный скролл, **прогресс-бар** при активном просмотре сторис  
- `Dashboard` — две колонки: `ChatPreview` / `RoomPreview`  
- `Feed` + `PostCard` — только **Comment** и **Share**, без лайков в UI  
- `BottomNav` — навигация  
- `useWebSocket` — переподключение, heartbeat

## Развёртывание (локально)

Из корня: `npm install` в `client` и `server`, затем `npm run dev` в корне (см. `package.json`) — параллельно клиент и сервер.
